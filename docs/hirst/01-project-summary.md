# Damien Hirst. 세계관의 연대기

> 데미언 허스트의 세계관이 어떤 대표 연대기로 구획되며 어떻게 발전해왔는지를, 연대기별 키워드 통계와 작품에 실제로 사용된 동물·곤충 생체/사체 도감으로 풀어내는 단일 스크롤 큐레이션.

## 배경 및 목적

### 왜 만드는가

- 기존 코드베이스는 "Mark Rothko의 감정선과 자주 사용한 색"이라는 색-감정 매핑 컨셉으로 만들어졌다. Hirst에 그 골격을 그대로 얹으면 작가의 본질(죽음·종교·과학·소비) 이 색채 분포로 환원되어버린다.
- Hirst를 다루려면 축이 달라야 한다: 색이 아니라 **세계관**, 감정선이 아니라 **연대기적 사상 발전**.
- 동시에 Hirst의 대중적 진입점 — "포름알데히드 상어, 박제, 5,000마리의 나비" — 라는 정량적 사실은 그 자체로 강력한 후크. 이를 별도 큐리오 섹션이 아닌 연대기 안에 정량 도감으로 통합한다.

### 기대 효과

- 대중에게는 "허스트가 왜 그런 작품을 만들었는가"를 시간 순으로 따라가는 서사 경험.
- 큐레이션 관점에서는 5축 키워드 택소노미(MORTALITY 외 4축)가 시간에 따라 어떻게 재가중되는지를 시각적으로 입증.
- 기존 Rothko 기반 컴포넌트(타임라인 캔버스, 도넛, 곡선)를 폐기하지 않고 변형 재사용하여 작업량 절감.

## 핵심 기능

| # | 기능 | 설명 | 우선순위 |
|---|------|------|---------|
| 1 | 연대기 세그먼트 타임라인 | 단일 수직 스크롤. Hirst의 작업 인생을 4~6개의 대표 연대기(Era)로 분절하여 세그먼트로 표현. 각 세그먼트에 진입하면 해당 시기의 핵심 명제·대표작·주요 사건이 함께 노출 | 필수 |
| 2 | 연대기별 키워드 통계 시각화 | 5축 택소노미(MORTALITY/SCIENCE_RELIGION/MARKET/MEDIA_AURA/MATERIAL)의 가중치가 연대기마다 어떻게 재분배되는지를 도넛/파이로 표현. 기존 ColorDonutChart의 변형 재사용 | 필수 |
| 3 | 생체·사체 정량 도감 | 작품에 실제 사용된 동물·곤충의 종/마리 수를 연대기 안에 인라인 노출. 절단 상어 1, 양 1, 나비 5,000+ 등 정량 사실을 명시. raw data는 사용자가 제공 | 필수 |
| 4 | 세계관 발전 곡선 | TimelineEmotionCurve를 변형하여 5축 강도의 시간축 곡선으로 재활용. 어떤 사상이 언제 부상/쇠퇴했는지의 경향선 | 필수 |
| 5 | 작품 상세 모달 | 작품 클릭 시 사용 매체·생체 종 수·해당 연대기 사상과의 연결 1~2줄을 띄움. 기존 ColorDetailModal 자리에 들어감 | 필수 |
| 6 | 다국어(KO/EN) | 기존 i18n 구조 유지. 연대기명·키워드·사상 명제는 모두 ko/en 동시 노출 | 필수 |
| 7 | 미니맵·앵커 내비 | 긴 스크롤에서 현재 연대기 표시 + 점프. 기존 TimelineMinimap 변형 | 선택 |

## 대상 사용자

- **주요 사용자**: 현대미술에 관심은 있으나 Hirst를 "상어 작가" 정도로만 아는 일반 관람객. 작가의 사상적 변천을 한 번에 파악하고 싶은 사람.
- **보조 사용자**: 큐레이터/미술교육자가 강의·전시 기획 시 Hirst 세계관을 빠르게 정리해 보여줄 수 있는 레퍼런스.

## 기술적 범위

### 포함

- React + Vite + MUI v7 기존 스택 유지.
- 단일 스크롤 SPA. 라우팅 추가 없음.
- 데이터: `hirst_works.json` / `hirst_events.json` / `hirst_keyword_taxonomy.json` 3종 활용 + 신규 `hirst_eras.json`(연대기 정의), `hirst_specimens.json`(생체·사체 도감) 추가.
- 시각화: 기존 `TimelineCanvas`/`TimelineAxis`/`ColorDonutChart`/`TimelineEmotionCurve`를 의미론적으로 재정의(색→키워드, 감정→사상강도)하여 변형 재사용.
- i18n KO/EN.

### 제외

- 작품의 색채 분석/색상 매핑 (Rothko 컨셉의 잔재. 완전 폐기).
- 사용자 입력·계정·저장 기능. 큐레이션 정적 사이트.
- 백엔드/DB 연동. 모든 데이터는 정적 JSON.
- AI 자동 분류·생성. 키워드 가중치는 사람이 사전 큐레이션.

### 제약사항

- `public/images/hirst/` 가 비어 있어 작품 이미지가 placeholder로 fallback 중. 이미지 자산 확보가 시각 완성도의 병목.
- 생체·사체 raw data는 사용자가 제공하기 전까지 Phase 2 데이터 모델에 포함시키되 인스턴스는 비워둠.
- 기존 `ruleRelationships.js` / 디자인 시스템 룰 4종(`code-convention`, `design-system`, `mui-grid-usage`, `directory-structure`) 준수.

## 성공 기준

- 첫 화면 진입 후 스크롤만으로 Hirst의 4~6개 연대기와 각 시기의 핵심 사상을 1분 안에 파악할 수 있다.
- 임의의 연대기 세그먼트에서 "이 시기에 가장 비중이 큰 키워드 축"과 "이 시기에 사용된 동물/곤충 종과 수"가 동시에 보인다.
- Rothko용 색-감정 컴포넌트가 의미론적으로 Hirst 컨셉으로 완전히 재정의되어, "색"이라는 단어가 본문에 등장하지 않는다.
